# AGENTS.md

本文件规定后续 Codex 或其他自动化工程助手修改本项目时必须遵守的规则。

## 项目定位

本项目是硕士论文实验工程，题目为“基于智能优化的自适应低秩大模型微调方法研究”。代码应优先保证实验可复现、结构清晰、配置可追踪，避免为了短期方便写成一次性脚本。

## 修改规则

1. 不要默认启动大模型训练。任何训练、搜索或评估脚本都必须提供 `--dry-run` 或等价保护，并在 README 中给出显式关闭 dry-run 的命令。
2. 所有脚本必须支持命令行参数，不能把模型名、数据路径、输出目录、学习率、rank 等关键参数硬编码死。
3. 新增 Python 代码应包含必要中文注释，注释解释实验意图、张量含义或非显然逻辑，不写空泛注释。
4. 配置优先放在 `configs/`，实验产物优先放在 `outputs/` 或 `experiments/`。
5. 不要把数据集、模型权重、adapter 权重、日志大文件提交到版本管理。
6. 修改 LoRA 或 QLoRA 逻辑时，必须保持 PEFT `LoraConfig` 与 `rank_pattern` 的兼容性。
7. 修改 PSO 搜索逻辑时，必须记录每个候选 rank pattern、随机种子、目标函数分数、可训练参数量估计和实验输出目录。
8. 新增实验命令后，必须同步更新 README 的完整流程命令。
9. 若引入新依赖，必须更新 `requirements.txt` 或 `requirements-qlora.txt`，并说明用途。
10. 不要重写用户已有实验结果。写入 `experiments/runs/` 或 `outputs/` 时应使用可区分的目录名。

## 推荐代码风格

- 包内公共逻辑放在 `src/adaptive_lora_thesis/`，命令行入口放在 `scripts/`。
- 数据格式统一为 JSONL，每行包含 `instruction`、`input`、`output`，可选 `system`。
- 与模型架构相关的 layer 名称集中在配置或工具函数中，不要散落在多个脚本里。
- 评估指标应写入 JSON，便于论文表格后处理。
- 随机过程必须暴露 `--seed` 参数。
